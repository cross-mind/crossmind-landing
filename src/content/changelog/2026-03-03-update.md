过去两天我们专注在让平台更稳定、更可靠上。如果你最近遇到过 Twitter/LinkedIn 集成意外断开，或者定时任务没按时运行，这次更新解决了这些问题。

## OAuth 集成现在更可靠了

我们修复了三个影响 OAuth 认证稳定性的核心问题：

**不再因为临时错误就断开连接**  
之前如果 Twitter API 返回 403 错误（通常是临时限流），我们会直接把整个集成标记为失效。现在系统能区分"真正的认证失败"（401）和"暂时的访问问题"（403）。这意味着你的社交媒体集成不会因为 API 短暂抽风就断开。

**并发请求不再打架**  
当多个任务同时使用同一个 OAuth token 时，之前会出现竞态问题导致 token 刷新失败。我们优化了 token 刷新的锁机制，现在可以安全地并发使用 Twitter 和 LinkedIn 集成。

**自动检测失效并提醒**  
当真的遇到认证问题（比如 token 被用户手动撤销），系统会自动标记集成为失效状态并通知你重新授权。不再静默失败。

## 定时任务终于准时了

修复了一个严重的 bug：使用"interval"类型的定时任务（比如"每2天"、"每12小时"）一直不会触发。问题出在数据库字段初始化和时间单位解析上。现在所有周期性任务都会按时执行。

如果你之前设置了 interval 任务但从来没跑过，现在应该能正常工作了。

## 其他改进

- **Artifact 系统更清晰**: 移除了 task_id 字段（因为一个 artifact 可以被多个任务使用），添加了刷新按钮，支持视频嵌入
- **邮件配置统一**: 解决了不同场景下邮件 URL 不一致的问题
- **代码架构简化**: 清理了 base agent 中的冗余组件，让系统更容易维护
